// Ground Rules — Read First

The 5 Rules of This Plan

  1. Never skip a day. Missing one day creates a two-day hole. If life happens, do 30 minutes minimum — don't do zero.
  2. Apply while you study. Start sending applications from Day 3. Interviews take 2–3 weeks to arrive. Apply now, prep simultaneously.
  3. Build, don't just read. Every AWS concept you learn — type it, deploy it, break it. Reading without doing = forgetting in 48 hours.
  4. Know what YOU built. Before any interview, write down exactly which components of your RAG project you personally owned. Never claim more, never claim less.
  5. Explain out loud every day. Pick one concept you studied and explain it out loud like you're in an interview. This is non-negotiable. Most candidates read everything but can't speak it.

Goal: Own Your Architecture
DAY 01
Monday
AWS Lambda
Lambda Deep Dive — How it Actually Works
  • Read: Lambda execution model — cold start vs warm start, execution environment lifecycle
  • Read: Concurrency types — reserved concurrency vs provisioned concurrency vs unreserved
  • Understand: What happens when 50 users hit Lambda simultaneously
  • Build: Write a simple Lambda function locally using AWS SAM CLI
  • Explain out loud: "How does Lambda scale when traffic spikes?"
⏱️ Morning: Concurrency docs | Evening: SAM CLI setup + first function
DAY 02
Tuesday
API Gateway
API Gateway — How it Connects to Lambda
  • Read: REST API vs HTTP API in API Gateway — differences, when to use which
  • Read: Integration types — Lambda proxy integration vs custom integration
  • Understand: Request/response flow from client → API GW → Lambda → response
  • Read: Throttling — rate limit vs burst limit, how 429 errors happen

⏱️ Morning: API GW docs | Evening: Wire Lambda + API GW + test with Postman
DAY 03
Wednesday
Lambda Authorizer

  • Read: Lambda Authorizer types — TOKEN vs REQUEST authorizer
  • Understand: JWT validation flow — how you verify Okta tokens in Lambda
  • Understand: Authorizer caching — TTL, why it matters, cache by header
  • Read: OAuth 2.0 flows — Client Credentials vs Authorization Code (know both)

⏱️ Morning: Authorizer docs | Evening: Applications + explain OAuth flow out loud

Aurora PostgreSQL + RDS Proxy + pgvector

  • Read: pgvector extension — HNSW vs IVFFlat indexes, when to use each
  • Understand: Cosine similarity vs L2 distance vs inner product in pgvector

  • Read: SQS Standard vs FIFO — differences, at-least-once vs exactly-once

Final Honest Note

  1. 21 days is enough — if you do the work every single day. The plan is designed for mid-company interviews, not FAANG. The bar is real-world problem solving, not algorithmic perfection.
  2. Your RAG experience is your moat. No one at a mid-company interview will have built what you've built at 2 years. Lead every conversation with it.
  3. Interviews will start arriving in Week 2. Don't panic — you're studying while the pipeline fills. That's the design.
  4. Come back and do mock interviews here after Week 1. You'll notice the difference immediately in how you answer technical questions.
  5. The goal is not perfection. The goal is to be better than the other candidate sitting in the same interview. 21 days of focused prep does that.
